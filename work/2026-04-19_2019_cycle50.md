## 1. Fleet Taxonomy (Derived from Active Fleet Repos)

From reading fleet-knowledge/ and active fleet repos, I derive the following taxonomy:

- **plato** – Training room infrastructure, LoRA export, self‑training systems.

- **agent‑runtime** – Agent execution environments, orchestration, messaging.
- **research‑trail** – Exploratory research, rabbit‑trail documents, concept proofs.
- **tool‑integration** – API wrappers, model benchmarks, external service adapters.
- **docs‑standards** – Documentation templates, README standards, style guides.
- **ops‑monitoring** – Health checks, service guards, logging, deployment scripts.
- **brand‑assets** – Logos, wordmarks, visual identity, architectural branding.

**Status Tags:**
- **active** – Actively developed, part of the fleet core.
- **archive** – Historical, not actively maintained but kept for reference.
- **stub** – Skeleton or placeholder, minimal content.
- **experiment** – One‑off test, proof‑of‑concept, not production‑ready.
- **deprecated** – Superseded, not recommended for use.
