## 1. Fleet Taxonomy (Derived from Active Fleet Repos)

From reading fleet-knowledge/ and active fleet repos, I derive the following taxonomy:

**Core Domains:**
- `plato` – Training room infrastructure (PLATO system)
- `ensign` – LoRA adapter export from room experience
- `holodeck` – MUD server with sentiment-aware NPCs
- `simulator` – Fleet simulation for training data
- `constraint-theory` – Geometric snapping/constraint solving
- `cudaclaw` – GPU-resident agent runtime
- `flux` – Self-assembling bytecode VM
- `shell` – Bootstrapping algorithms (crab trap)
- `deadband` – Protocol implementation (P0/P1/P2)
- `zeroclaw` – Agent runtime (git repo as shell)

**Support Categories:**
- `tool` – Utility, CLI, helper scripts
- `research` – Experiment, rabbit‑trail, concept exploration
- `archive` – Inactive, deprecated, historical
- `docs` – Documentation, specifications, memory
- `asset` – Brand, logo, visual identity
- `test` – Test suite, validation, benchmarking

**Status Tags:**
- `active` – Actively developed, part of fleet workflow
- `stale` – No recent commits, may need revival
- `abandoned` – No longer maintained, candidate for archive
- `seed` – Early concept, minimal code
- `example` – Demonstration, tutorial, reference
