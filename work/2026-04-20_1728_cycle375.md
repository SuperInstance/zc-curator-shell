## 1. Fleet Taxonomy (Derived from Active Fleet Repos)

From reading fleet-knowledge/ and active fleet repos, I derive this working taxonomy:

**Core Domains:**
- **plato** – Training room infrastructure (presets, LoRA export, tile handling)
- **holodeck** – Simulation environments (MUD server, NPCs, sentiment)
- **fleet-simulator** – Multi-agent simulation for training data generation
- **constraint-theory** – Geometric/relational constraint solving (snapping, alignment)
- **cudaclaw** – GPU-resident agent runtime (kernel-level execution)
- **flux-runtime** – Self-assembling bytecode VM (dynamic code generation)
- **shell** – Bootstrapping/capture algorithms (crab trap, classification, scoring)
- **ensign** – Specialist model exports (LoRA adapters, fine‑tuned weights)
- **vessel** – Agent containers/orchestration (Docker, Kubernetes, service management)
- **research** – Experimental trails, prototypes, academic exploration
- **tool** – Utilities, CLI tools, devops scripts, helpers
- **docs** – Documentation, specifications, architecture notes
- **legacy** – Deprecated, archived, or inactive projects
- **stub** – Placeholder, empty, or just‑started repos

**Tags (additional context):**
- `rust`, `python`, `cpp`, `docker`, `kubernetes`
- `training`, `inference`, `simulation`, `geometry`, `agent`
- `active`, `wip`, `stable`, `experimental`, `abandoned`
