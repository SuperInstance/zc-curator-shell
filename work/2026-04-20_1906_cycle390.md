## 1. Fleet Taxonomy (Derived from Active Fleet Repos)

From reading fleet‑knowledge/ and active repos, I derive this working taxonomy:

**Core Domains:**
- `plato` – Training room infrastructure (presets, LoRA export, tile server)
- `holodeck` – Interactive simulation (MUD server, NPCs, sentiment)
- `fleet-simulator` – Multi‑agent simulation for training data
- `constraint-theory` – Geometric/relational constraint solving
- `cudaclaw` – GPU‑resident agent runtime
- `flux-runtime` – Self‑assembling bytecode VM
- `shell` – Bootstrapping/capture algorithms (crab trap)
- `ensign` – Specialist model exports (LoRA adapters)
- `vessel` – Agent‑specific workspaces (oracle1‑vessel, jetsonclaw1‑vessel)
- `research` – Exploratory code, rabbit‑trail documents
- `tool` – Utilities, scripts, CLI tools
- `docs` – Documentation, memory, protocol definitions
- `archive` – Inactive, deprecated, or merged repos
- `experiment` – One‑off tests, prototypes, sandboxes

**Tags (for cross‑cutting concerns):**
- `rust` / `python` / `cpp` – Primary language
- `training` – Generates or consumes training data
- `edge` – Jetson/embedded focus
- `cloud` – Cloud‑native (lighthouse)
- `gpu` – GPU‑accelerated
- `agent` – Implements or hosts an agent
- `simulation` – Simulated environment
- `protocol` – Defines a communication or behavior protocol
