## 1. Understand Fleet Taxonomy

From reading fleet-knowledge/ and existing repo patterns, I identify these core categories:

### Core Fleet Components
- **plato-***: Training room systems (plato-torch, plato-ensign)
- **holodeck-***: Interactive simulation environments (holodeck-rust)
- **fleet-***: Fleet-wide tools and simulations (fleet-simulator)
- **constraint-***: Geometric/constraint systems (constraint-theory-core)
- **cudaclaw**: GPU-resident agent runtime
- **flux-***: VM/runtime systems (flux-runtime)

### Infrastructure & Tools
- **zeroclaw-***: Agent bootstrapping and shells
- **service-***: Monitoring and operations
- **docs-***: Documentation and knowledge
- **tools-***: Development utilities

### Research & Experiments
- **research-***: Exploratory projects
- **experiment-***: Prototypes and tests
- **demo-***: Demonstrations and examples

### Legacy & Inactive
- **archive-***: Historical or deprecated
- **stale-***: No recent activity
- **fork-***: External code forks
