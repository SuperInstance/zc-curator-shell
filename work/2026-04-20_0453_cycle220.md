- **holodeck** – Simulation environments (MUD server, NPCs, sentiment‑aware)

- **cudaclaw** – GPU‑resident agent runtime (CUDA, Triton)

- **shell‑bootstrap** – Bootstrapping algorithms (classify→score→complicate→capture)
- **deadband‑protocol** – P0/P1/P2 decision framework implementations
- **agent‑runtime** – Agent execution frameworks (not core fleet tech)
- **research** – Exploratory code, experiments, rabbit‑trail docs
- **tooling** – Development utilities, scripts, CI/CD
- **docs** – Documentation, memory, brand assets
- **archive** – Inactive, deprecated, or prototype‑only repos

**Tags (additional context):**
- `rust` / `python` / `cuda` – Primary language
- `training` – Produces or consumes training tiles
- `simulation` – Simulated environment
- `edge` – Jetson/embedded target
- `cloud` – Cloud‑native deployment
- `plugin` – Extensible plugin architecture
- `lora` – LoRA adapter generation/management
- `mirror‑play` – I2I iterative training
