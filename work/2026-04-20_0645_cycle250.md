## 1. Fleet Taxonomy (Derived from Active Fleet Repos)

From reading fleet-knowledge/ and active fleet repos, I derive the following taxonomy:

**Core Domains:**
- **plato** – Training room infrastructure, LoRA export, self‑training systems
- **holodeck** – Multi‑user dungeon (MUD) servers, sentiment‑aware NPCs, interactive simulation
- **fleet‑simulator** – Fleet‑level simulation for generating training data
- **constraint‑theory** – Geometric snapping, constraint solving, CAD‑like logic
- **cudaclaw** – GPU‑resident agent runtime, low‑latency inference
- **flux‑runtime** – Self‑assembling bytecode VM, dynamic code generation
- **shell‑bootstrapping** – Algorithms to capture, classify, score, and complicate external intelligence
- **deadband‑protocol** – P0/P1/P2 decision framework, safety‑first navigation
- **agent‑orchestration** – Multi‑agent frameworks, communication protocols, role‑based systems
- **tool‑integration** – API wrappers, model connectors, external service adapters
- **research‑trails** – Exploratory code, experiments, one‑off proofs of concept
- **documentation** – READMEs, protocol specs, architecture descriptions, training materials
- **brand‑assets** – Logos, wordmarks, visual identity, marketing materials

**Tags (for finer granularity):**
- `training` – Involves model training, fine‑tuning, or data generation
- `simulation` – Simulates environments, agents, or interactions
- `runtime` – Execution environment for agents or models
- `protocol` – Defines communication or decision standards
- `research` – Experimental, not yet integrated into core
- `tool` – Utility, helper, or integration component
- `docs` – Purely documentation
- `legacy` – Old version, deprecated, or inactive
- `wip` – Work in progress, incomplete
