## 1. Fleet Taxonomy (Derived from Active Fleet Repos)

From reading fleet-knowledge/ and active fleet repos, I derive the following taxonomy:

**Core Domains:**
- **plato** – Training room infrastructure (presets, LoRA export, tile handling)
- **holodeck** – Interactive simulation environments (MUD server, NPCs)
- **fleet-simulator** – Multi-agent simulation for training data generation
- **constraint-theory** – Geometric/relational constraint solving (snapping, rules)
- **cudaclaw** – GPU-resident agent runtime (CUDA kernels, low-latency execution)
- **flux-runtime** – Self-assembling bytecode VM (dynamic code generation)
- **shell** – Bootstrapping/capture algorithms (classify→score→complicate→capture)
- **ensign** – Specialist model exports (LoRA adapters, fine‑tuned weights)
- **vessel** – Agent containers/communication (bottles, message passing)
- **tools** – Development utilities (CLI, API wrappers, benchmarks)
- **research** – Exploratory code, experiments, rabbit‑trail documents
- **docs** – Documentation, memory, protocol specifications
- **brand** – Visual identity, logos, wordmarks, brand assets
- **legacy** – Archived, deprecated, or inactive projects

**Tags (for cross‑cutting concerns):**
- `rust` | `python` | `cpp` – Primary language
- `training` – Generates or consumes training data
- `simulation` – Simulates environments or agents
- `agent-runtime` – Hosts or executes agents
- `geometry` – Spatial/geometric operations
- `gpu` – GPU‑accelerated computation
- `protocol` – Communication or data‑exchange standards
- `cli` – Command‑line interface
- `api` – Web/network API
- `archive` – Read‑only, historical value only
