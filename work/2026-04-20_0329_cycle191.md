- **simulator** – Fleet simulation, training data generation (fleet-simulator)

- **cudaclaw** – GPU-resident agent runtime, CUDA kernels
- **flux** – Self-assembling bytecode VM, runtime systems (flux-runtime)
- **shell** – Bootstrapping, agent capture, classification algorithms
- **deadband** – Deadband Protocol implementation, P0/P1/P2 logic
- **zeroclaw** – Agent framework, boot camp, tile submission
- **docs** – Documentation, memory, vision, protocol specs
- **tools** – Utilities, scripts, service management (service-guard.sh)
- **research** – Rabbit trails, experiments, exploratory code
- **legacy** – Old prototypes, deprecated, not actively maintained
- **stub** – Empty, placeholder, or very minimal repos
- **external** – Forks of external projects, third-party code
