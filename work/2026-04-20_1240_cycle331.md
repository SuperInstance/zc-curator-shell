## 1. Fleet Taxonomy (Derived from Active Fleet Repos)

From reading fleet-knowledge/ and active fleet repos, I derive this taxonomy:

**Core Domains:**
- **plato** – Training room systems, LoRA export, self‑training infrastructure.
- **holodeck** – Multi‑user dungeon (MUD) servers, sentiment‑aware NPCs, interactive simulation.
- **fleet‑simulator** – Fleet‑level simulation for generating training data.
- **constraint‑theory** – Geometric snapping, constraint solving, spatial reasoning.
- **cudaclaw** – GPU‑resident agent runtime, low‑latency inference.
- **flux‑runtime** – Self‑assembling bytecode VM, dynamic code generation.
- **shell‑bootstrap** – Bootstrapping algorithms for capturing external intelligence.
- **deadband‑protocol** – Implementation of P0/P1/P2 decision protocol.
- **agent‑runtime** – Agent execution environment, message passing, tool use.
- **research‑trail** – Exploratory code, experiments, one‑off proofs of concept.
- **tool‑integration** – Wrappers for external APIs (Groq, Anthropic, etc.).
- **docs‑brand** – Documentation, branding assets, memory files.
- **legacy‑archive** – Old prototypes, deprecated code, historical reference.

**Tags (for finer granularity):**
- `training` – Involves model training or fine‑tuning.
- `simulation` – Simulated environments or synthetic data generation.
- `rust` – Primary language is Rust.
- `python` – Primary language is Python.
- `cuda` – GPU‑accelerated CUDA code.
- `mud` – Multi‑user dungeon / game server.
- `protocol` – Communication or decision protocols.
- `runtime` – Execution environment or VM.
- `research` – Experimental, not production‑ready.
- `tool` – Integration with external service.
- `docs` – Documentation only.
- `archive` – No longer actively developed.
