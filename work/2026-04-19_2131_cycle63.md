## 1. Fleet Taxonomy (Derived from Active Fleet Repos)

From reading fleet-knowledge/ and active fleet repos, I derive the following taxonomy:

**Core Domains:**
- **plato** – Training room infrastructure, LoRA export, self‑training algorithms
- **holodeck** – MUD server, sentiment‑aware NPCs, interactive simulation
- **fleet‑simulator** – Multi‑agent simulation for generating training data
- **constraint‑theory** – Geometric snapping, spatial reasoning, constraint solving
- **cudaclaw** – GPU‑resident agent runtime, low‑latency execution
- **flux‑runtime** – Self‑assembling bytecode VM, dynamic code generation
- **shell‑bootstrapping** – Algorithms to capture external intelligence (classify→score→complicate→capture)
- **deadband‑protocol** – P0/P1/P2 decision framework, negative‑space mapping
- **agent‑orchestration** – Multi‑agent coordination, communication protocols
- **tool‑integration** – API wrappers, model providers, external service adapters
- **research‑trails** – Exploratory code, experiments, one‑off proofs of concept
- **documentation** – READMEs, protocols, memory, training materials
- **brand‑assets** – Logos, wordmarks, visual identity, lighthouse/crab imagery

**Tags (for finer granularity):**
- `training` – Produces or consumes training data
- `simulation` – Synthetic environment or agent‑in‑the‑loop
- `runtime` – Execution environment for agents
- `geometry` – Spatial reasoning, snapping, constraints
- `gpu` – CUDA, GPU‑resident code
- `rust` – Primary language is Rust
- `python` – Primary language is Python
- `typescript` – Primary language is TypeScript
- `legacy` – Old version, deprecated, or inactive
- `archive` – Historical record only, not for active development
- `experimental` – High‑risk, unproven, research‑grade
- `production` – Stable, used in active fleet operations
- `tool` – CLI utility, helper script, service guard
