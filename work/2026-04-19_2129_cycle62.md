## 1. Fleet Taxonomy (Derived from Active Fleet Repos)

From reading fleet-knowledge/ and active fleet repos, I derive the following taxonomy:

**Core Domains:**
- **plato** – Training room infrastructure, LoRA export, self‑training algorithms.
- **holodeck** – Multi‑user dungeon (MUD) servers, sentiment‑aware NPCs, interactive simulation.
- **fleet‑simulator** – Fleet‑level simulation for generating training data.
- **constraint‑theory** – Geometric snapping, constraint solving, spatial reasoning.
- **cudaclaw** – GPU‑resident agent runtime, low‑latency inference.
- **flux‑runtime** – Self‑assembling bytecode VM, dynamic code generation.
- **shell‑bootstrap** – Bootstrapping algorithms for capturing external intelligence.
- **deadband‑protocol** – Implementation of the P0/P1/P2 decision framework.
- **agent‑orchestration** – Multi‑agent coordination, communication protocols.
- **tools‑config** – Model APIs, tool definitions, runtime configurations.
- **research‑trails** – Exploratory research, concept proofs, rabbit‑hole documents.
- **docs‑brand** – Documentation, branding assets, memory files.
- **legacy‑abandoned** – Inactive, unmaintained, or deprecated projects.

**Tags (for finer granularity):**
- `training` – Involves model training, LoRA adapters, data generation.
- `simulation` – Simulated environments, synthetic data.
- `runtime` – Execution environment, VM, inference engine.
- `geometry` – Spatial math, constraints, snapping.
- `protocol` – Communication or decision protocols.
- `orchestration` – Multi‑agent coordination.
- `research` – Experimental, not yet production.
- `docs` – Documentation only.
- `tooling` – Developer tools, scripts, utilities.
- `deprecated` – No longer active.
