From reading fleet-knowledge/ and active fleet repos, the following taxonomy emerges:

- **holodeck** – Simulation environments (MUD server, NPCs, sentiment-aware worlds)

- **cudaclaw** – GPU-resident agent runtime (kernel-level execution)
- **flux-runtime** – Self-assembling bytecode VM (modular execution)
- **shell** – Bootstrapping/capture algorithms (crab trap, classification, scoring)
- **ensign** – Specialist model exports (LoRA adapters, fine-tuned weights)
- **vessel** – Agent communication & state (bottles, message passing, memory)
- **deadband** – Protocol implementation (P0/P1/P2 decision layers)
- **tools** – Development utilities (CLI, API wrappers, benchmarking)
- **research** – Exploratory code, experiments, rabbit trails
- **docs** – Documentation, specifications, architecture notes
- **brand** – Visual identity, logos, wordmarks, brand assets
- **legacy** – Archived, deprecated, or inactive projects

**Tags (for cross-cutting concerns):**
- `rust` – Implemented in Rust
- `python` – Implemented in Python
- `torch` – Uses PyTorch
- `training` – Generates or uses training data
- `simulation` – Simulates environments or agents
- `gpu` – GPU-accelerated or resident
- `edge` – Edge computing deployment
- `cloud` – Cloud-native or cloud services
- `agent` – Multi-agent system component
- `protocol` – Communication or decision protocol
- `visual` – Visual output (UI, graphics, rendering)
- `cli` – Command-line interface tool
- `api` – API server or client
- `experimental` – High-risk, unstable, or proof-of-concept
